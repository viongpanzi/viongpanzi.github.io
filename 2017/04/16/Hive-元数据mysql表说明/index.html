Hive 元数据mysql表说明

发表于 2017-04-16

工作过程中需要对Hive的元数据进行操作，对元数据需要很熟悉，因此整理了一下元数据中用到的表，记录一下。现在先整理了一下MySQL中的每一个表的字段的含义如下：

BUCKETING_COLS

Field Type Null Key Default Extra Comment
SD_ID bigint(20) NO PRI NULL
BUCKET_COL_NAME varchar(256) YES NULL
INTEGER_IDX int(11) NO PRI NULL

CDS

Field Type Null Key Default Extra Comment
CD_ID bigint(20) NO PRI NULL

COLUMNS_V2

Field Type Null Key Default Extra Comment
CD_ID bigint(20) NO PRI NULL
COMMENT varchar(256) YES NULL
COLUMN_NAME varchar(128) NO PRI NULL
TYPE_NAME varchar(4000) NO NULL
INTEGER_IDX int(11) NO NULL

COMPACTION_QUEUE

Field Type Null Key Default Extra Comment
CQ_ID bigint(20) NO PRI NULL
CQ_DATABASE varchar(128) NO NULL
CQ_TABLE varchar(128) NO NULL
CQ_PARTITION varchar(767) YES NULL
CQ_STATE char(1) NO NULL
CQ_TYPE char(1) NO NULL
CQ_WORKER_ID varchar(128) YES NULL
CQ_START bigint(20) YES NULL
CQ_RUN_AS varchar(128) YES NULL

COMPLETED_TXN_COMPONENTS

Field Type Null Key Default Extra Comment
CTC_TXNID bigint(20) YES NULL

CTC_TABLE varchar(128) YES NULL
CTC_PARTITION varchar(767) YES NULL

DATABASE_PARAMS

Field Type Null Key Default Extra Comment
DB_ID bigint(20) NO PRI NULL 关联的数据库ID
PARAM_KEY varchar(180) NO PRI NULL 参数Key
PARAM_VALUE varchar(4000) YES NULL 参数值

该表存储在创建数据库时，使用WITH DBPROPERTIES (property_name=property_value, …)指定的参数。
